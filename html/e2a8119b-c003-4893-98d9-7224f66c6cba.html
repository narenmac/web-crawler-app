    本文档已翻译为多种语言，仅供参考之用。原始官方版本（英文版）可在以下位置找到： https://www.icann.org/privacy/policy

    [基于先前版本隐私政策的带修订标记版本] [PDF, 201 KB]

    互联网名称与数字地址分配机构（ICANN 组织）尊重个人信息的隐私性以及对其进行的保护。本政策（隐私政策）和 Cookie 政策规定了 ICANN 组织（就本政策而言，包括其附属机构公共技术标识符 [PTI]，统称为"我们"、"我们的"或"我方"）及其管理或发起的项目对个人提供的或从个人处收集的电子信息的使用和处理方式，以及个人对信息收集和使用的不同选择。本隐私政策适用于通过使用各种电子方式收集和使用个人信息，包括通过网站的任何移动版本、移动应用程序、电子邮件以及链接至本隐私政策的其他电子服务（统称为"平台"）。术语"用户"或"您"是指：(i) 向我们提供其自己的个人信息或代表他人向我们提供此类其他人的个人信息的个人；以及 (ii) 由个人代表其提供他人的个人信息的任何组织。

    使用平台即代表您（包括您代表的任何实体或个人）同意更新后的本隐私政策的条款。如果您不同意本隐私政策中所述的惯例，请不要向我们提供个人信息或与平台交互。

    本隐私政策补充但不取代我们就任何独特项目提供的任何其他隐私政策或声明。

    如果您对本隐私政策或者我们（作为数据控制人）对您个人信息的处理有任何疑问，请随时联系我们：[email protected]。

    本隐私政策涵盖以下关键主题：

    1. 定义
    2. 个人信息的收集
    3. 个人信息的使用
    4. 数据保留
    5. 个人信息的共享
    6. 国际传输
    7. 信息安全和完整性
    8. 请勿追踪
    9. 数据主体权利的行使
    10. 未成年人
    11. 隐私政策的修订
    12. 疑问

    1. 定义

      "Cookie"是在您访问互联网上的某些网站时存储在您计算机上用于识别您的计算机的小文本文件。要了解更多有关 Cookie 以及我们平台上的某些功能和特性的信息，请访问 Cookie 政策。

      "数据控制人"是指单独或与其他方联合确定处理个人信息的目的和方式的自然人或法人、公共权威机构、机关或其他主体。

      "数据处理人"是指代表数据控制人处理个人信息的自然人或法人、公共权威机构、机关或任何其它主体。

      "非识别信息"是指可能与某个人相关，但单凭该信息却不能识别此人的信息。 例如，网页信息便属于非识别信息，再如与其他个人信息无关联的性别、年龄、 城市和物理位置等信息。

      "个人信息"是指适用法律下规定的与某个已识别或可识别的自然人相关的信息，例如姓名、家庭地址、电话号码、照片、录影或电子邮件地址。

      "平台"的定义请参阅我们的服务条款。

    2. 个人信息的收集

      本隐私政策适用于与用户或平台使用有关的以下类型电子信息：

      • 提交给我们的个人信息：此类信息可能是个人联系方式、识别信息和位置信息，例如用户可能就我们的任何服务或项目（包括公众意见提交、项目或服务申请、社群调查、电子邮件列表订阅、一般问询、活动注册或在线会议）提交的姓名、家庭或工作地址、个人或商业电子邮件地址、联系方式（如家庭、传真机或移动电话号码）、性别、年龄或出生日期、职称、居住国、学校、所属机构或组织、用户名、域名（若可识别个人身份）、用户的照片或图像、政府发布的身份证号码或护照详细信息或紧急联络人详细信息。此类信息还包括在平台上提供给我们的、构成适用法律下规定的个人资料的信息。

      • 从第三方处获得的个人信息：此类信息可包括用于身份验证或处理提交给我们的某项请求或申请的姓名和地址等个人信息。

      • 财务信息：此类信息可能包括参与我们的任何服务或项目时所需要的或生成的财务账户信息或财务交易信息。

      • 自动记录的使用信息：访问和使用平台时，平台可自动保存用户的浏览器或设备发送的识别或非识别信息。此类信息可能包括 IP 地址、浏览器类型、互联网服务提供商 (ISP)、日期/时间戳、浏览的页面、操作系统、移动设备信息、点击流信息、偏好信息和其他类似信息。

      • Cookie 和其他追踪技术：使用平台时，Cookie 和其他用户追踪技术设备可储存在用户的计算机上。关于我们使用 Cookie 和其他追踪技术的更多信息，请查看我们的 Cookie 政策。您认可使用本隐私政策和 Cookie 政策中所述的此类追踪设备，并同意将此类追踪设备存储在您的计算设备中。

      • 录音、照片和文稿：我们收集并保存公开和非公开会议的录音、照片和/或文稿，包括通过在线会议平台进行收集，其中会包含您的个人信息（如果您是与会者）。对这些会议的录音、照片和文稿进行处理是基于我们促进问责制、公开性和透明度的合法利益。这些会议的录音、照片和文稿以及其中包含的任何个人信息都可能允许公开访问。当我们主办的会议和培训需要进行录制时，我们将在每次会议开始前告知您和其他参与者这一情况。当我们录制会议时，任何录音和相关文稿中包含的您的个人信息将根据本隐私政策进行处理。

        本政策不适用于使用非我们提供或支持的服务的任何会议，除非该服务另有说明。请注意，选择不使用我们提供或支持的服务提供、主办或参与会议的个人，应全权负责确保与会者在知情的情况下同意录音，并确保全面遵守收集、记录和处理个人信息的所有法律规定。

    3. 个人信息的使用

      我们将仅出于公平、合法和正当的理由处理个人信息，包括用于：

      • 改善我们的平台性能、内容及服务；
      • 进行信息分析、合规性审计、数据度量，开发新服务，保护平台以及我们的任何服务或项目的完整性；
      • 创建匿名数据集以改善平台性能，或创建与我们的服务或项目相关的匿名数据集；
      • 回复用户的问题、意见、咨询、反馈，包括与我们的服务或项目有关的或涉及注册管理运行机构、认证注册服务机构和数据托管代理提供的服务的问题、意见、咨询、反馈；
      • 根据我们的职责协助政策讨论、开发活动以及活动的执行；
      • 管理我们的任何服务、活动、项目、会议和平台的参与或注册，包括发送与此类参与或注册相关的电子通信；并管理和报告这些参与、互动和关系；
      • 就我们的活动向参与者提供差旅补助；
      • 根据适用法律规定的数据保留限制，保留用户参与的存档记录用于历史记录和研究目的；
      • 在用户就我们的任何服务或项目提交付款信息的情况下，处理付款和完成用户与我们之间的交易；
      • 报销与我们的任何会议或其他活动或项目有关的费用；
      • 履行对用户和其他第三方的相关合同义务；
      • 管理用户对我们的任何服务或项目提出的申请；
      • 在平台上公布用户的个人信息，例如姓名和组织或所属机构；
      • 为了其他人的利益并为了遵守我们的问责制和透明度原则，公布用户的意见或反馈以及用户的姓名；
      • 注册并提供参与我们会议的机会，可能涉及处理用户的饮食限制或残疾信息以提供特殊的膳宿；
      • 加快用户的签证申请并获得批准，以便参加我们的会议；
      • 向同一会议的其他参与者分享用户自愿选择提供的基本参与者信息（姓名、组织、所属机构）或其他信息；
      • 提供有关我们的项目、活动和服务交付的最新资讯、新闻以及其他信息； 以及
      • 提供用户请求的信息或服务；

      在适用法律要求的情况下，我们还将在征得您同意后处理您的个人信息，以便向您发送简讯，并向您提供涉及新闻、最新资讯的营销信息以及有关我们项目和活动的其他信息。

      最后，我们将处理个人信息以遵守适用法律、条例、法规、法院指令以及执法机构要求。

      我们不会将个人信息用于本隐私政策规定之外的任何其他目的。

      一般情况下，我们会将非识别信息用于与个人信息相同的目的，并用于改善用户服务、存储用户的平台偏好、追踪平台使用趋势、改进平台的性能、帮助保持个人和设备身份验证、平台安全以及测量电子通信的有效性。

    4. 数据保留

      我们将按照我们的一般存档惯例并按照法律规定或在法律允许的情况下，保留存储在我们服务器上的某些个人信息。

    Domain Name System
    Internationalized Domain Name ,IDN,"IDNs are domain names that include characters used in the local representation of languages that are not written with the twenty-six letters of the basic Latin alphabet ""a-z"". An IDN can contain Latin letters with diacritical marks, as required by many European languages, or may consist of characters from non-Latin scripts such as Arabic or Chinese. Many languages also use other types of digits than the European ""0-9"". The basic Latin alphabet together with the European-Arabic digits are, for the purpose of domain names, termed ""ASCII characters"" (ASCII = American Standard Code for Information Interchange). These are also included in the broader range of ""Unicode characters"" that provides the basis for IDNs. The ""hostname rule"" requires that all domain names of the type under consideration here are stored in the DNS using only the ASCII characters listed above, with the one further addition of the hyphen ""-"". The Unicode form of an IDN therefore requires special encoding before it is entered into the DNS. The following terminology is used when distinguishing between these forms: A domain name consists of a series of ""labels"" (separated by ""dots""). The ASCII form of an IDN label is termed an ""A-label"". All operations defined in the DNS protocol use A-labels exclusively. The Unicode form, which a user expects to be displayed, is termed a ""U-label"". The difference may be illustrated with the Hindi word for ""test"" — परीका — appearing here as a U-label would (in the Devanagari script). A special form of ""ASCII compatible encoding"" (abbreviated ACE) is applied to this to produce the corresponding A-label: xn--11b5bs1di. A domain name that only includes ASCII letters, digits, and hyphens is termed an ""LDH label"". Although the definitions of A-labels and LDH-labels overlap, a name consisting exclusively of LDH labels, such as""icann.org"" is not an IDN."
